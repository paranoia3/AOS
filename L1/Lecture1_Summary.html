Lecture 1 — Introduction to Operating Systems

Summary & explanation of the core concepts

1. Что такое операционная система

ОС — это программа-посредник между пользователем и аппаратурой компьютера. Её главные задачи:

  • Выполнять программы пользователя и упрощать решение задач
  • Делать компьютер удобным в использовании
  • Эффективно использовать аппаратные ресурсы

ОС как распределитель ресурсов

Управляет всеми ресурсами и принимает решения при конфликтующих запросах для справедливого и эффективного использования.

ОС как программа управления

Контролирует выполнение программ, чтобы предотвратить ошибки и неправильное использование компьютера.

Ядро (kernel) — это единственная программа, работающая постоянно. Всё остальное — это либо системные программы, либо прикладные программы.

2. Структура компьютерной системы

Компьютерная система состоит из четырёх компонентов:

  • Аппаратура — CPU, память, устройства ввода/вывода
  • Операционная система — координирует использование аппаратуры
  • Прикладные программы — текстовые редакторы, компиляторы, браузеры, СУБД, игры
  • Пользователи — люди, машины, другие компьютеры

3. Запуск и работа компьютера

Bootstrap-программа

При включении компьютера запускается загрузочная программа, хранящаяся в ROM/EPROM (прошивка). Она инициализирует систему и загружает ядро ОС, после чего передаёт ему управление.

4. Прерывания и подсистема ввода/вывода

ОС управляется прерываниями (interrupt-driven). Прерывание передаёт управление сервисной процедуре через вектор прерываний.

  • Hardware interrupt — от устройства
  • Trap / Exception — программное прерывание (ошибка или системный вызов)
  • DMA — прямой доступ устройств к памяти; одно прерывание на блок, а не на байт

Контроллер устройства имеет локальный буфер, обменивается данными с памятью, а по окончании операции сигнализирует CPU прерыванием.

5. Хранилище данных и его иерархия

Основная единица — bit. 1 байт = 8 бит. Слово (word) — собственный размер данных архитектуры (обычно 64 бита).

Основная память (RAM)

Единственное крупное хранилище, к которому CPU обращается напрямую. Произвольный доступ, обычно энергозависимая.

Вторичное хранилище

HDD, SSD — большой нелетучий объём. SSD быстрее HDD и набирает популярность.

Кэширование — копирование информации из медленного хранилища в быстрое. Применяется на всех уровнях: аппаратура, ОС, ПО. Главные задачи — размер кэша и политика замены.

6. Архитектура компьютерных систем

  • Однопроцессорные — один процессор общего назначения + специализированные
  • Мультипроцессорные (параллельные) — выше производительность, экономия масштаба, отказоустойчивость
    • Asymmetric MP — каждый процессор имеет свою задачу
    • Symmetric MP — все процессоры выполняют любые задачи
  • Кластерные системы — несколько систем, разделяющих хранилище через SAN, для высокой доступности и HPC

7. Структура ОС: многозадачность

Multiprogramming (пакетная обработка)

В памяти держится несколько задач; пока одна ждёт I/O, CPU переключается на другую — CPU не простаивает.

Timesharing / Multitasking

CPU переключается между задачами так часто, что пользователь может взаимодействовать с каждой. Время отклика должно быть менее 1 секунды. Используется планирование CPU, swapping и виртуальная память.

Режимы работы ОС

Аппаратура поддерживает двойной режим (mode bit):

  • User mode — обычный режим выполнения программ
  • Kernel mode — привилегированный режим для системных операций

Системный вызов переключает режим в kernel; возврат — обратно в user. Таймер предотвращает бесконечные циклы — по истечении он генерирует прерывание, возвращая управление ОС.

8. Управление процессами

Процесс — программа в исполнении, единица работы в системе. Программа — пассивна, процесс — активен.

ОС отвечает за:

  • Создание и удаление процессов
  • Приостановку и возобновление
  • Синхронизацию между процессами
  • Межпроцессное взаимодействие (IPC)
  • Разрешение тупиков (deadlock handling)

Однопоточный процесс имеет один program counter; многопоточный — по одному на поток.

9. Управление памятью и хранилищем

Memory Management

Отслеживание занятых областей, решение о загрузке/выгрузке процессов, выделение и освобождение памяти.

Storage Management

Абстракция физического носителя в логическое понятие — файл. Управление каталогами, правами доступа, резервным копированием.

Подсистема ввода/вывода

Скрывает особенности устройств. Отвечает за буферизацию, кэширование, спулинг и общий интерфейс драйверов.

10. Защита и безопасность

  • Protection — контроль доступа процессов и пользователей к ресурсам
  • Security — защита от внутренних и внешних атак (вирусы, DoS, кража учётных данных)

Каждому пользователю присваивается user ID; группы — group ID. Privilege escalation позволяет временно повысить права.

11. Среды вычислений

Традиционная

Автономные ПК, объединённые через Интернет, защищённые межсетевыми экранами.

Мобильная

Смартфоны, планшеты с GPS, гироскопами, беспроводной связью. Лидеры — iOS и Android.

Распределённая

Сети: LAN, WAN, MAN, PAN. ОС создаёт иллюзию единой системы через TCP/IP.

Клиент-серверная

Серверы обслуживают запросы клиентов (compute-server, file-server).

Peer-to-Peer

Узлы равноправны: каждый может быть клиентом и сервером одновременно. Примеры: Napster, Gnutella, Skype (VoIP).

Виртуализация

Запуск гостевых ОС внутри host-ОС с помощью VMM (например, VMware, VirtualBox). Используется для разработки, тестирования и в дата-центрах.

Облачные вычисления

Логическое расширение виртуализации. Типы: Public, Private, Hybrid. Модели: SaaS, PaaS, IaaS.

Встраиваемые системы реального времени

Самая распространённая форма компьютеров. Real-time ОС имеют жёсткие временны́е ограничения — корректная работа только при их соблюдении.

12. Open-Source ОС

ОС с открытым исходным кодом — противоположность закрытым проприетарным системам. Начало положено FSF с лицензией GPL (copyleft). Примеры: GNU/Linux, BSD UNIX (включая ядро Mac OS X).

Ключевые выводы

  • ОС — посредник между пользователем и аппаратурой; ядро — её сердце
  • ОС управляется прерываниями и работает в привилегированном режиме
  • Иерархия хранилища и кэширование — основа производительности
  • Многозадачность и виртуальная память позволяют эффективно использовать ресурсы
  • Современные среды: мобильные, распределённые, виртуальные, облачные

1. What Is an Operating System

An OS is a program that acts as an intermediary between the user and the computer hardware. Its main goals:

  • Execute user programs and make problem-solving easier
  • Make the computer convenient to use
  • Use hardware resources efficiently

OS as Resource Allocator

Manages all resources and arbitrates conflicting requests for efficient, fair use.

OS as Control Program

Controls program execution to prevent errors and improper use of the computer.

The kernel is the single program running at all times. Everything else is either a system program (ships with the OS) or an application program.

2. Computer System Structure

A computer system has four components:

  • Hardware — CPU, memory, I/O devices
  • Operating system — coordinates hardware use
  • Applications — word processors, compilers, browsers, databases, games
  • Users — people, machines, other computers

3. Computer Startup & Operation

Bootstrap Program

Loaded at power-up from ROM/EPROM (firmware). It initializes the system, loads the OS kernel, and transfers control to it.

4. Interrupts & I/O

An OS is interrupt-driven. An interrupt transfers control to a service routine via the interrupt vector.

  • Hardware interrupt — raised by a device
  • Trap / Exception — software-generated (error or syscall)
  • DMA — devices transfer to memory directly; one interrupt per block instead of per byte

Each device controller has a local buffer, exchanges data with main memory, and signals the CPU on completion.

5. Storage & Its Hierarchy

The basic unit is a bit. 1 byte = 8 bits. A word is the architecture's native data size (typically 64 bits).

Main Memory (RAM)

The only large storage the CPU accesses directly. Random access, typically volatile.

Secondary Storage

HDDs and SSDs — large, non-volatile capacity. SSDs are faster and increasingly popular.

Caching — copying information from slower to faster storage. Applied at every level (hardware, OS, software). Key design issues: cache size and replacement policy.

6. Computer System Architecture

  • Single-processor — one general-purpose CPU plus special-purpose ones
  • Multiprocessor (parallel) — higher throughput, economy of scale, fault tolerance
    • Asymmetric MP — each processor has a specific task
    • Symmetric MP — every processor performs all tasks
  • Clustered systems — multiple machines share storage via a SAN for high availability and HPC

7. OS Structure: Multitasking

Multiprogramming (Batch)

Several jobs are kept in memory; when one waits for I/O, the CPU switches to another — the CPU is never idle.

Timesharing / Multitasking

The CPU switches between jobs so quickly users can interact with each. Response time should be < 1 second. Requires CPU scheduling, swapping, and virtual memory.

OS Operation Modes

Hardware supports dual-mode operation (mode bit):

  • User mode — normal program execution
  • Kernel mode — privileged mode for system operations

A system call switches to kernel mode; returning resets to user mode. A timer prevents infinite loops — when it expires it generates an interrupt and returns control to the OS.

8. Process Management

A process is a program in execution — the unit of work in the system. A program is passive; a process is active.

The OS is responsible for:

  • Creating and deleting processes
  • Suspending and resuming them
  • Process synchronization
  • Inter-process communication (IPC)
  • Deadlock handling

A single-threaded process has one program counter; a multi-threaded one has a PC per thread.

9. Memory & Storage Management

Memory Management

Tracks which parts of memory are in use, decides what to swap in/out, and allocates/frees memory.

Storage Management

Abstracts physical media into the logical concept of a file. Handles directories, access control, and backup.

I/O Subsystem

Hides device peculiarities. Provides buffering, caching, spooling, and a uniform device-driver interface.

10. Protection & Security

  • Protection — controls process/user access to resources
  • Security — defends against internal and external attacks (viruses, DoS, identity theft)

Each user has a user ID; groups have a group ID. Privilege escalation temporarily grants elevated rights.

11. Computing Environments

Traditional

Stand-alone PCs interconnected via the Internet, protected by firewalls.

Mobile

Smartphones and tablets with GPS, gyroscopes, wireless connectivity. Leaders: iOS and Android.

Distributed

Networks: LAN, WAN, MAN, PAN. The OS creates the illusion of a single system over TCP/IP.

Client-Server

Servers respond to client requests (compute-server, file-server).

Peer-to-Peer

All nodes are peers — each can act as client and server. Examples: Napster, Gnutella, Skype (VoIP).

Virtualization

Guest OSes run within a host OS through a VMM (VMware, VirtualBox). Used in development, testing, and data centers.

Cloud Computing

A logical extension of virtualization. Types: Public, Private, Hybrid. Models: SaaS, PaaS, IaaS.

Real-Time Embedded Systems

The most prevalent form of computers. Real-time OSes have strict time constraints — correct behavior requires they be met.

12. Open-Source Operating Systems

OSes shipped in source form — the opposite of closed proprietary systems. Started by the FSF with the GPL copyleft license. Examples: GNU/Linux, BSD UNIX (including the core of Mac OS X).

Key Takeaways

  • The OS mediates between user and hardware; the kernel is its core
  • OSes are interrupt-driven and run in a privileged mode
  • Storage hierarchy and caching are the foundation of performance
  • Multitasking and virtual memory enable efficient resource use
  • Modern environments: mobile, distributed, virtual, cloud
